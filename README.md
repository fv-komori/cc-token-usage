# cc-token-usage

Claude Code のローカルログからトークン使用量を集計し、**Amazon Bedrock（従量課金）で使ったと仮定した場合のコスト**を試算する単一ファイルの Python スクリプトです。

- 依存ライブラリなし（Python 3 標準ライブラリのみ）
- `~/.claude/projects/**/*.jsonl` を読むだけ。**データを外部に送信しません**
- モデル別（Opus / Fable / Sonnet / Haiku）に単価を分けて集計

> ⚠️ あくまで概算ツールです。Claude Code をサブスクリプションで使っている場合の実請求額とは別物で、「同じトークン量を Bedrock の従量課金で使ったらいくらか」という **参考値** を出します。

## 必要環境

- Python 3.8+
- Claude Code のローカルログ（`~/.claude/projects/` 配下の `*.jsonl`）

## 使い方

```bash
# 本日の時間帯別（トークンのみ）
python3 tok.py

# 指定時刻以降（トークン + コスト試算）
python3 tok.py "2026-06-26 09:00"

# 範囲指定（トークン + コスト試算）
python3 tok.py "2026-06-26 09:00" "2026-06-26 12:00"

# 日別サマリ（トークン + コスト試算）
python3 tok.py daily
```

日時はローカルタイムゾーンの `"YYYY-MM-DD HH:MM"` 形式で指定します。

### どこからでも使えるようにする（任意）

```bash
ln -s "$(pwd)/tok.py" ~/tok.py        # python3 ~/tok.py で実行
# または PATH の通った場所へ
ln -s "$(pwd)/tok.py" /usr/local/bin/tok   # tok daily などで実行
```

## 仕組み

1. `~/.claude/projects/**/*.jsonl` を全走査し、`message.usage` を持つ行を集計
2. `message.id` などで重複（resume で同じ応答が複数ファイルに現れるケース）を除外
3. `message.model` からモデルを判定し、モデル別の単価でコストを算出
4. `<synthetic>` などの非課金メッセージは除外

## 単価について

Bedrock の標準（グローバル / オンデマンド）単価は Anthropic API の標準レートと同額です。本スクリプトは以下を内蔵しています（USD / 100万トークン）。

| モデル | 入力 | 出力 | 5分書込 (×1.25) | 1時間書込 (×2) | 読込 (×0.1) |
|--------|-----:|-----:|----------------:|---------------:|-----------:|
| Fable 5        | $10 | $50 | $12.50 | $20 | $1.00 |
| Opus 4.8 / 4.7 | $5  | $25 | $6.25  | $10 | $0.50 |
| Sonnet 4.6     | $3  | $15 | $3.75  | $6  | $0.30 |
| Haiku 4.5      | $1  | $5  | $1.25  | $2  | $0.10 |

- キャッシュ書込は、ログの `usage.cache_creation` に記録された **5分キャッシュ（×1.25）／1時間キャッシュ（×2）の内訳** をそのまま使って計算します。内訳が無い古い記録は5分扱いにフォールバックします。
- 地域固定エンドポイントは +10%、Batch API は 50% 引き（末尾に目安を表示）。
- 単価は変わることがあります。最新情報は下記を確認してください。
  - [Anthropic / Claude Platform 料金](https://platform.claude.com/docs/en/about-claude/pricing)
  - [Amazon Bedrock 料金](https://aws.amazon.com/bedrock/pricing/)

単価を変更したい場合は `tok.py` 冒頭の `RATES` を編集します。
